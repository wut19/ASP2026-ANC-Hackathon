### 3. 物理架构与系统建模

在开始编码之前，你需要理解耳机声学空间内的信号流：

![ANC 系统示意图](assets/anc.png)

- **参考麦克风（Reference Mic, `x[n]`）：** 位于耳机**外壳**上。由于外壳的物理隔离，它只采集外部环境噪声。
- **初级通路（Primary Path, `P(z)`）：** 环境噪声穿透耳机外壳和耳垫到达耳膜的物理路径。
  - **目标信号 `d[n]`：** 定义为 `d[n] = x[n] * p[n]`，即关闭降噪时你在耳内**实际听到的噪声**。
- **耳机扬声器（Speaker, `y[n]`）：** 播放由你的算法生成的"反相噪声"。
- **次级通路（Secondary Path, `S(z)`）：** 从 DSP 输出开始，经过 DAC、功放、扬声器振膜振动、耳道空气传播，最终到达误差麦克风的**完整路径**。
- **误差麦克风（Error Mic, `e[n]`）：** 位于耳道**内部**，测量消噪后的残余噪声：

```math
e[n] = d[n] + s[n] * y[n]
```

### 5. 两阶段挑战

> **⚠️ 重要工作流程：先推导，再编码**
>
> 每个阶段都要求你**先完成数学公式推导，再动手写代码**。具体来说：
> 1. 借助 AI 从目标函数出发，逐步推导出完整的算法更新公式；
> 2. 确认推导无误后，将公式转化为 Python 代码实现。
>
> 报告中需要包含完整的推导过程。**直接跳过推导、让 AI 生成代码将不被接受。**

#### **Phase I：理想耳机（LMS 验证）**

- **场景：** 假设次级通路瞬时且无失真（`S(z) = 1`）。
- **数据：** 加载 `x_ref.npy` 和 `d_target.npy`。
- **任务：**
  1. **公式推导：** 借助 AI，从 MSE 代价函数出发，推导标准 LMS 自适应滤波器的权重更新公式（包括误差信号、梯度估计、权重迭代三个步骤）。将推导过程记录到报告中。
  2. **代码实现：** 基于推导结果，编写 Python 代码实现 LMS 滤波器，尝试消除 `d[n]`。
  3. **分析调优：** 记录收敛过程，找到使 MSE 衰减最快的最优步长 `μ`。
- **输出：** 你的代码必须产生一个形状为 `(N,)` 的数组 `e_lms` —— 残余误差信号。

#### **Phase II：振荡现实（FxLMS 进化）**

- **场景：** 引入真实的次级通路脉冲响应 `s_path_impulse.npy`。你会发现 Phase I 的代码因为相位延迟而导致系统发散或"啸叫"。
- **任务：**
  1. **问题诊断与公式推导：** 借助 AI，分析为什么 `S(z)` 的延迟会导致标准 LMS 发散。然后从修正后的代价函数出发，推导 **Filtered-x LMS (FxLMS)** 的完整权重更新公式。将推导过程记录到报告中。
  2. **代码实现：** 基于推导结果，实现 FxLMS 算法。
    - **核心逻辑：** 在权重更新前，将参考信号 `x[n]` 通过次级通路的估计 `Ŝ(z)` 进行滤波。
  3. **结果验证：** 恢复系统稳定性，计算最终的**噪声衰减（SNR 提升）**，单位 dB。
- **输出：** 你的代码必须产生一个形状为 `(N,)` 的数组 `e_fxlms` —— FxLMS 残余误差信号。

### 6. `anc_algorithm.py` 的工作方式

模板文件 `anc_algorithm.py` 结构如下：

```
┌─────────────────────────────────────┐
│  0. 加载数据（已填好）               │
├─────────────────────────────────────┤
│  Phase I：在此编写 LMS 代码         │
│  ↓ 输出：e_lms                     │
│  ── 自动评估 ────────────────────── │
│  • 打印衰减量（dB）                 │
│  • 保存 phase1_result.png           │
├─────────────────────────────────────┤
│  Phase II：在此编写 FxLMS 代码      │
│  ↓ 输出：e_fxlms                   │
│  ── 自动评估 ────────────────────── │
│  • 打印衰减量（dB）                 │
│  • 保存 phase2_result.png           │
│  • 保存 spectrum_comparison.png     │
│  • 播放降噪前后的音频对比            │
└─────────────────────────────────────┘
```

**你只需要填写算法代码。** 评估部分会自动运行，包括：

1. 计算并显示每个阶段的**噪声衰减量（dB）**
2. 生成**时域对比图**（`phase1_result.png`、`phase2_result.png`）
3. 生成**频谱对比图**（`spectrum_comparison.png`）—— ANC 前后的噪声功率谱密度
4. 播放**音频 Demo** —— ANC 前后人耳听感对比（需要 `sounddevice`）
